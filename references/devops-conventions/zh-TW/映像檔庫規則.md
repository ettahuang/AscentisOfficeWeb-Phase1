# 映像檔庫規則

## Registry 資訊

- Registry URL: https://docker.owgps.net/
- 專案映像檔名稱: at-devops-playground-demo

## 映像檔命名規範

### 格式

```bash
docker.owgps.net/at-devops-playground-demo:<tag>
```

### 標籤策略

1. **開發建置**

```bash
dev-$(tmp=${CI_PIPELINE_CREATED_AT//-/};echo ${tmp:0:8})-$CI_PIPELINE_ID-$CI_COMMIT_REF_SLUG
```

範例: `dev-20240327-123456-feature-demo`

2. **正式建置**

```bash
prod-$(tmp=${CI_PIPELINE_CREATED_AT//-/};echo ${tmp:0:8})-$CI_PIPELINE_ID-$CI_COMMIT_REF_SLUG
```

範例: `prod-20240327-123456-main`

3. **Latest 標籤**

```bash
latest-<ENVIRONMENT>
```

範例:
- `latest-jpcg`
- `latest-twt1p`
- `latest-kltwp`

## 映像檔操作


### Push 指令

```bash
docker push docker.owgps.net/at-devops-playground-demo:<tag>
```

## 生命週期管理

### 保留政策

| 標籤類型 | 保留期間 | 自動清理 |
|----------|----------|----------|
| `dev-xxx` | 2 週 | 是 |
| `prod-xxx` | 永久 | 否 |
| `latest-xxx` | 每環境單一標籤 | 否 |

### 政策細節

1. **開發標籤 (`dev-xxx`)**
   - 保留期間：2 週
   - 啟用自動清理
   - 用於開發和測試

2. **正式標籤 (`prod-xxx`)**
   - 不自動刪除
   - 永久保留
   - 用於正式環境部署

3. **Latest 標籤 (`latest-xxx`)**
   - 每個環境單一標籤
   - 每次成功部署時更新
   - 不自動刪除

## 技術限制

| 限制項目 | 上限 |
|----------|------|
| Image Tag 最大長度 | 128 bytes |
| CI_COMMIT_REF_SLUG 最大長度 | 63 bytes |

## 部署流程

### Deploy Job 工作流程

1. 發送部署開始通知
2. 為映像檔加上正式標籤
3. 為映像檔加上 latest 標籤
4. 更新部署
5. 發送部署完成通知

### AWS CLI 標籤操作

```bash
# 正式標籤
aws ecr tag xxx:$SOURCE_TAG xxx:prod-$TAG

# Latest 標籤
aws ecr tag xxx:$SOURCE_TAG xxx:latest-$TAG
```

## 最佳實踐

1. **部署時始終使用特定標籤**
   - 避免在正式環境使用 `latest` 標籤
   - 使用版本化標籤以便追蹤

2. **在映像檔標籤中包含建置中繼資料**
   - 建置時間戳記
   - Git commit SHA
   - Pipeline ID

3. **推送前執行安全掃描**
   - 執行漏洞掃描
   - 檢查是否有暴露的密鑰

4. **維護自訂標籤的文件**
   - 記錄任何非標準的標籤格式
   - 保持命名規範一致

## GitLab CI/CD 範例

```yaml
variables:
  IMAGE_NAME: docker.owgps.net/at-devops-playground-demo
  DEV_TAG: dev-${CI_PIPELINE_CREATED_AT}-${CI_PIPELINE_ID}-${CI_COMMIT_REF_SLUG}
  PROD_TAG: prod-${CI_PIPELINE_CREATED_AT}-${CI_PIPELINE_ID}-${CI_COMMIT_REF_SLUG}

build:
  stage: build
  script:
    - docker build -t ${IMAGE_NAME}:${DEV_TAG} .
    - docker push ${IMAGE_NAME}:${DEV_TAG}
  only:
    - branches

deploy-prod:
  stage: deploy
  script:
    - docker tag ${IMAGE_NAME}:${DEV_TAG} ${IMAGE_NAME}:${PROD_TAG}
    - docker tag ${IMAGE_NAME}:${DEV_TAG} ${IMAGE_NAME}:latest-prod
    - docker push ${IMAGE_NAME}:${PROD_TAG}
    - docker push ${IMAGE_NAME}:latest-prod
  only:
    - main
  when: manual
```
