# 層級職責定義

> 本文詳細定義 Clean Architecture 各層的職責範圍，並透過 PaymentService 專案的實際程式碼說明各層的正確實作方式。

---

## 架構分層總覽

```
┌─────────────────────────────────────────────────────────────┐
│  Presentation Layer (Controllers/)                          │
│  職責：HTTP 請求處理、回應格式化                               │
├─────────────────────────────────────────────────────────────┤
│  Application Layer (Services/)                              │
│  職責：用例協調、介面定義、業務流程編排                         │
├─────────────────────────────────────────────────────────────┤
│  Domain Layer (Models/Domain/)                              │
│  職責：核心業務邏輯、領域實體、業務規則                         │
├─────────────────────────────────────────────────────────────┤
│  Infrastructure Layer (Repositories/)                        │
│  職責：資料存取、外部服務整合、技術實作細節                     │
└─────────────────────────────────────────────────────────────┘
```

## Domain Layer（領域層）

### 職責定義

| 職責 | 說明 |
|------|------|
| **業務邏輯** | 封裝核心業務規則和計算 |
| **領域實體** | 定義業務概念的資料結構 |
| **值物件** | 定義無身份識別的值類型 |
| **業務驗證** | 驗證業務規則的合法性 |

### 專案位置

```
PaymentService/Models/Domain/
├── Customer.cs                    # 客戶實體
├── CustomerContext.cs             # 客戶上下文（組合物件）
├── DepositChannel.cs              # 存款通道實體
├── DepositChannels.cs             # 存款通道集合
├── DepositChannelsByDepositType.cs # 依類型分組的結果
├── DepositLimitInfo.cs            # 存款限額資訊
├── DepositLimitInfos.cs           # 存款限額集合
└── DepositUnLimitInfo.cs          # 無限額資訊
```

### 完整範例：DepositChannel

```csharp
namespace PaymentService.Models.Domain
{
    /// <summary>
    /// 存款通道領域模型，封裝通道的業務規則
    /// </summary>
    public class DepositChannel
    {
        // ==================== 屬性定義 ====================
        // 使用 init setter 確保不可變性
        public CurrencyEnum Currency { get; init; }
        public DepositTypeEnum DepositType { get; init; }
        public List<int> VipLevels { get; init; }
        public List<int> ExcludeTags { get; init; }
        public int PointLevel { get; init; }
        public decimal MinAmount { get; init; }
        public decimal MaxAmount { get; init; }
        public int Sort { get; init; }

        // ==================== 業務邏輯方法 ====================

        /// <summary>
        /// 判斷客戶是否適用此存款通道
        /// </summary>
        public bool IsSupported(CustomerContext custContext)
        {
            return custContext.IsCurrencyOf(Currency)
                   && custContext.IsReachedPointLevel(PointLevel)
                   && custContext.IsVipLevelInScope(VipLevels)
                   && !custContext.IsTagInScope(ExcludeTags);
        }

        /// <summary>
        /// 判斷金額範圍是否與限額有交集
        /// </summary>
        public bool IsOverlappingWith(DepositLimitInfo depositLimitInfo)
        {
            if (depositLimitInfo == null)
                throw new ArgumentNullException(nameof(depositLimitInfo));

            if (!IsValidAmountRange() || !depositLimitInfo.IsValidAmountRange())
                return false;

            return MinAmount <= depositLimitInfo.MaxAmount
                   && MaxAmount >= depositLimitInfo.MinAmount;
        }

        /// <summary>
        /// 套用金額限制，回傳新實例（維持不可變性）
        /// </summary>
        public DepositChannel WithAmountLimit(DepositLimitInfo depositLimitInfo)
        {
            if (depositLimitInfo == null)
                throw new ArgumentNullException(nameof(depositLimitInfo));

            var newMin = Math.Max(MinAmount, depositLimitInfo.MinAmount);
            var newMax = Math.Min(MaxAmount, depositLimitInfo.MaxAmount);

            if (newMin > newMax)
                throw new InvalidOperationException(
                    "No overlapping range exists. Call IsOverlappingWith() before WithAmountLimit().");

            // 回傳新實例，而非修改現有實例
            return new DepositChannel(
                Currency, DepositType, ExcludeTags, VipLevels, PointLevel,
                newMin, newMax, Sort);
        }

        // ==================== 私有驗證方法 ====================

        private bool IsValidAmountRange()
        {
            return MinAmount <= MaxAmount;
        }
    }
}
```

### Domain Layer 規則

| 規則 | 說明 |
|------|------|
| **不可變性** | 使用 `init` setter，修改時回傳新實例 |
| **無外部依賴** | 不依賴 Repository、HttpClient、DbContext |
| **業務語言命名** | 使用 `IsSupported()`、`WithAmountLimit()` 等業務術語 |
| **自我驗證** | 在方法內進行業務規則驗證 |

### Domain Layer 禁止事項

```csharp
// ❌ 禁止：注入外部服務
public class DepositChannel
{
    private readonly IDepositRepository _repo;  // 錯誤！
    private readonly ILogger _logger;            // 錯誤！
}

// ❌ 禁止：依賴 DbModel
public class DepositChannel
{
    public DbModel.DepositChannel DbModel { get; set; }  // 錯誤！
}

// ❌ 禁止：使用可變 setter
public class DepositChannel
{
    public decimal MinAmount { get; set; }  // 應該用 init
}
```

## Application Layer（應用層）

### 職責定義

| 職責 | 說明 |
|------|------|
| **用例協調** | 協調多個 Repository 完成業務流程 |
| **介面定義** | 定義 Repository 和 Service 的介面 |
| **流程編排** | 決定呼叫順序和資料流向 |
| **委託業務邏輯** | 將業務運算委託給 Domain Model |

### 專案位置

```
PaymentService/Services/
├── ICustomerRepository.cs         # Repository 介面
├── IDepositRepository.cs          # Repository 介面
├── IDepositQueryService.cs        # Service 介面
└── DepositQueryService.cs         # Service 實作
```

### 完整範例：DepositQueryService

```csharp
namespace PaymentService.Services
{
    /// <summary>
    /// 存款查詢服務，協調 Repository 以完成存款通道查詢用例
    /// </summary>
    public class DepositQueryService : IDepositQueryService
    {
        private readonly ICustomerRepository _customerRepository;
        private readonly IDepositRepository _depositRepository;

        public DepositQueryService(
            ICustomerRepository customerRepository,
            IDepositRepository depositRepository)
        {
            _customerRepository = customerRepository;
            _depositRepository = depositRepository;
        }

        /// <summary>
        /// 取得客戶可用的存款通道（依存款類型分組）
        /// </summary>
        public async Task<DepositChannelsByDepositType> GetDepositChannelsByDepositTypeAsync(int custId)
        {
            // 1. 協調：取得客戶上下文
            var custContext = await _customerRepository.GetCustContextAsync(custId);

            // 2. 協調：取得存款通道
            var siteId = custContext.Customer.SiteId;
            var depositChannels = await _depositRepository.GetDepositChannelsAsync(siteId);

            // 3. 委託：業務邏輯交給 Domain Model
            return depositChannels.GroupByDepositType(custContext);
        }
    }
}
```

### Application Layer 規則

| 規則 | 說明 |
|------|------|
| **只做協調** | 不包含業務邏輯計算 |
| **依賴介面** | 依賴 Repository 介面，非實作 |
| **委託 Domain** | 業務運算委託給 Domain Model |
| **定義介面** | Repository 介面定義於此層 |

## Infrastructure Layer（基礎設施層）

### 職責定義

| 職責 | 說明 |
|------|------|
| **資料存取** | 實作 Repository 介面，執行資料庫操作 |
| **模型轉換** | 將 DbModel 轉換為 Domain Model |
| **外部服務整合** | 整合第三方 API、訊息佇列等 |
| **橫切關注點** | 透過 Decorator 實作快取、日誌等 |

### 專案位置

```
PaymentService/Repositories/
├── CustomerRepository.cs          # Repository 實作
├── CustomerRepositoryDecorator.cs # 快取 Decorator
├── DepositRepository.cs           # Repository 實作
└── DepositRepositoryDecorator.cs  # 快取 Decorator

PaymentService/Models/DbModel/
├── Customer.cs                    # 資料庫映射模型
├── DepositChannel.cs              # 資料庫映射模型
└── DepositLimitInfo.cs            # 資料庫映射模型
```

### 完整範例：CustomerRepository

```csharp
namespace PaymentService.Repositories
{
    /// <summary>
    /// 客戶資料存取實作
    /// </summary>
    public class CustomerRepository : ICustomerRepository
    {
        private readonly MySqlClient _mainDbClient;

        public CustomerRepository(IEnumerable<MySqlClient> mySqlClients)
        {
            var sqlClients = mySqlClients.ToList();
            _mainDbClient = sqlClients.First(client => client.DataBase == DataBase.Main);
        }

        /// <summary>
        /// 取得客戶資料
        /// </summary>
        public async Task<Customer> GetCustInfoAsync(int custId)
        {
            // 1. 執行資料庫查詢，取得 DbModel
            var dbCust = await _mainDbClient.QueryFirstAsync<DbModel.Customer>(
                "Customer_Get",
                new { CustId = custId });

            // 2. 透過 CreateDomain() 轉換為 Domain Model
            return dbCust?.CreateDomain();
        }

        /// <summary>
        /// 取得客戶積分等級
        /// </summary>
        public async Task<int> GetPointLevelAsync(int custId)
        {
            return await _mainDbClient.ExecuteScalarAsync<int>(
                "Member_PointLevel_Get",
                new { CustId = custId });
        }

        /// <summary>
        /// 取得完整客戶上下文
        /// </summary>
        public async Task<CustomerContext> GetCustContextAsync(int custId)
        {
            var customer = await GetCustInfoAsync(custId);
            if (customer == null)
                throw new CustomerNotFoundException(custId);

            var pointLevel = await GetPointLevelAsync(custId);
            return new CustomerContext(customer, pointLevel);
        }
    }
}
```

### DbModel 與 CreateDomain 模式

```csharp
namespace PaymentService.Models.DbModel
{
    /// <summary>
    /// 資料庫映射模型（對應資料庫欄位）
    /// </summary>
    public class Customer
    {
        public int? SysId { get; init; }
        public int CustId { get; init; }
        public int SiteId { get; init; }
        public byte VipLevel { get; init; }
        public string Tags { get; init; }     // JSON 字串
        public int CurrencyId { get; init; }   // int 對應 Enum

        /// <summary>
        /// 轉換為 Domain Model
        /// </summary>
        public Domain.Customer CreateDomain()
        {
            return new Domain.Customer
            {
                CustId = CustId,
                SiteId = SiteId,
                VipLevel = VipLevel,
                Tags = Tags.ParseJson<List<int>>(),        // JSON 轉 List
                Currency = (CurrencyEnum)CurrencyId        // int 轉 Enum
            };
        }
    }
}
```

### Decorator 實作快取

```csharp
namespace PaymentService.Repositories
{
    /// <summary>
    /// 客戶 Repository 的快取 Decorator
    /// </summary>
    public class CustomerRepositoryDecorator : ICustomerRepository
    {
        private readonly ICustomerRepository _customerRepository;
        private readonly ICacheHelper _cacheHelper;

        public CustomerRepositoryDecorator(
            ICustomerRepository customerRepository,
            ICacheHelper cacheHelper)
        {
            _customerRepository = customerRepository;
            _cacheHelper = cacheHelper;
        }

        public async Task<Customer> GetCustInfoAsync(int custId)
        {
            return await _cacheHelper.TryGetOrCreateAsync(
                $"CustInfo:{custId}",           // Cache Key
                TimeSpan.FromMinutes(1),         // TTL
                async () => await _customerRepository.GetCustInfoAsync(custId)
            );
        }

        public async Task<CustomerContext> GetCustContextAsync(int custId)
        {
            return await _cacheHelper.TryGetOrCreateAsync(
                $"CustomerContext:{custId}",
                TimeSpan.FromMinutes(1),
                async () => await _customerRepository.GetCustContextAsync(custId)
            );
        }
    }
}
```

### Infrastructure Layer 規則

| 規則 | 說明 |
|------|------|
| **實作介面** | 實作 Application Layer 定義的介面 |
| **回傳 Domain Model** | 使用 `CreateDomain()` 轉換 |
| **Decorator 分離** | 快取、日誌透過 Decorator 實作 |
| **只處理技術細節** | 不包含業務邏輯判斷 |

## Presentation Layer（展示層）

### 職責定義

| 職責 | 說明 |
|------|------|
| **HTTP 處理** | 處理 HTTP Request/Response |
| **輸入驗證** | 使用 Data Annotations 驗證輸入 |
| **格式轉換** | 將 Domain Model 轉換為 API Response |
| **路由定義** | 定義 API 端點 |

### 專案位置

```
PaymentService/Controllers/
└── CustomerController.cs

PaymentService/Models/Payload/
└── GetDepositOptionsRequest.cs
```

### 完整範例：Controller

```csharp
namespace PaymentService.Controllers
{
    [ApiController]
    [Route("api/[controller]")]
    public class CustomerController : ControllerBase
    {
        private readonly IDepositQueryService _depositQueryService;

        public CustomerController(IDepositQueryService depositQueryService)
        {
            _depositQueryService = depositQueryService;
        }

        /// <summary>
        /// 取得客戶可用的存款選項
        /// </summary>
        [HttpPost("GetDepositOptions")]
        public async Task<IActionResult> GetDepositOptions(
            [FromBody] GetDepositOptionsRequest request)
        {
            // 1. 呼叫 Service（不處理業務邏輯）
            var result = await _depositQueryService
                .GetDepositChannelsByDepositTypeAsync(request.CustId);

            // 2. 回傳統一格式
            return Ok(ApiResult.Success(result));
        }
    }
}
```

## 各層職責對比總結

| 層級 | 應該做 | 不應該做 |
|------|--------|----------|
| **Domain** | 業務邏輯、驗證規則、不可變性 | 資料存取、HTTP、快取、日誌 |
| **Application** | 協調 Repository、定義介面、委託 Domain | 業務邏輯、資料存取、快取 |
| **Infrastructure** | 資料存取、模型轉換、Decorator | 業務邏輯、HTTP 處理 |
| **Presentation** | HTTP 處理、輸入驗證、回應格式化 | 業務邏輯、資料存取 |

## ✅ Review Checklist

### Domain Layer 檢查

- [ ] 使用 `init` setter 而非 `set`？
- [ ] 業務邏輯方法是否在 Domain Model 內？
- [ ] 是否無外部依賴（Repository、HttpClient）？
- [ ] 方法命名是否使用業務語言？

### Application Layer 檢查

- [ ] Service 是否只做協調？
- [ ] 業務邏輯是否委託給 Domain Model？
- [ ] Repository 介面是否定義於此層？
- [ ] 是否依賴介面而非實作？

### Infrastructure Layer 檢查

- [ ] Repository 是否回傳 Domain Model？
- [ ] 是否使用 `CreateDomain()` 進行轉換？
- [ ] 快取是否透過 Decorator 實作？
- [ ] 是否只處理技術細節？

### Presentation Layer 檢查

- [ ] Controller 是否只處理 HTTP？
- [ ] 輸入驗證是否使用 Data Annotations？
- [ ] 是否呼叫 Service 而非直接呼叫 Repository？
- [ ] 是否使用統一的回應格式（ApiResult）？

## ✅ 新人常見踩雷點

### 1. 業務邏輯放錯層

```csharp
// ❌ 錯誤：業務邏輯在 Controller
public class CustomerController
{
    public async Task<IActionResult> GetChannels(int custId)
    {
        var channels = await _repo.GetChannels();
        // 這是業務邏輯，應該在 Domain Model！
        var filtered = channels.Where(c => c.VipLevels.Contains(vipLevel));
        return Ok(filtered);
    }
}

// ✅ 正確：業務邏輯在 Domain Model
public class DepositChannels
{
    public DepositChannels FilterSupported(CustomerContext custContext)
    {
        var filtered = Items.Where(c => c.IsSupported(custContext));
        return new DepositChannels(filtered);
    }
}
```

```csharp
// ❌ 錯誤：回傳 DbModel
public async Task<DbModel.Customer> GetCustInfoAsync(int custId)
{
    return await _db.QueryFirstAsync<DbModel.Customer>(...);
}

// ✅ 正確：回傳 Domain Model
public async Task<Customer> GetCustInfoAsync(int custId)
{
    var dbModel = await _db.QueryFirstAsync<DbModel.Customer>(...);

}
```

## ✅ TL / Reviewer 檢查重點

### 職責越界審查

| 發現問題 | 可能原因 | 建議處理 |
|----------|----------|----------|
| Controller 超過 50 行 | 業務邏輯混入 | 抽取至 Service/Domain |
| Service 有 Where/Select | 業務邏輯未委託 | 移至 Domain Model |
| Repository 有 if-else 業務判斷 | 職責混淆 | 移至 Domain Model |
| Domain Model 有 `_repo` 欄位 | 依賴錯誤 | 移除外部依賴 |

### 依賴方向審查

- [ ] Domain Layer 是否有 `using PaymentService.Repositories`？
- [ ] Application Layer 是否有 `using PaymentService.Models.DbModel`？
- [ ] Controller 是否直接依賴 Repository 實作？

### 命名審查

- [ ] Domain Model 方法是否使用業務語言？
- [ ] 介面是否以 `I` 開頭？
- [ ] DbModel 與 Domain Model 是否在正確命名空間？
